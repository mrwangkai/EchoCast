### Request 4: Toast Notifications System
**User asked:** Implement toast notifications for bookmark actions in two parts - create ToastView component, then integrate into EpisodePlayerView without breaking sheet anchoring

**Part 1 - Created ToastView.swift:**
```swift
struct ToastMessage: Equatable {
    let message: String
    let icon: String
}

struct ToastView: View {
    let toast: ToastMessage

    var body: some View {
        HStack(spacing: 8) {
            Image(systemName: toast.icon)
            Text(toast.message)
        }
        .foregroundColor(.white)
        .padding(.horizontal, 16)
        .padding(.vertical, 10)
        .background(Color.black.opacity(0.8))
        .cornerRadius(20)
        .shadow(color: .black.opacity(0.25), radius: 8, x: 0, y: 4)
    }
}
```

**Key learning:** Toast wrapper ZStack caused sheet anchoring issue - must add overlay inside existing ZStack.

### Request 10: Fix `.tabBarMinimizeBehavior(.onScrollDown)` on Home Tab (MAIN FIX)
**User asked:** At commit 815f1e2, `.tabBarMinimizeBehavior(.onScrollDown)` was working on the Home tab. Find what changed between 815f1e2 and now that could affect HomeView or ContentView

**Investigation process:**
1. Ran `git diff 815f1e2 HEAD -- EchoNotes/Views/HomeView.swift`
2. Ran `git diff 815f1e2 HEAD -- EchoNotes/ContentView.swift`
3. Compared TabView structure and HomeView navigation approach

**Key findings from diffs:**

**HomeView.swift changes between 815f1e2 and HEAD:**
- Added `@Binding var selectedTab: Int`
- Changed from `.navigationBarHidden(true)` to `.navigationTitle("EchoCast")` + `.navigationBarTitleDisplayMode(.large)`
- Added `.toolbar { }` block with buttons
- Moved header from inline HStack to system navigation bar

**ContentView.swift changes:**
- Tab structure changed (added PodcastDiscoveryView, reordered tabs)
- `.tabViewBottomAccessory(isEnabled:)` condition changed from `player.showMiniPlayer` to `player.currentEpisode != nil`
- Added `@Namespace` for matched geometry effect

**Root cause identified:**
The `.navigationBarTitleDisplayMode(.large)` creates a collapsing navigation bar that competes for scroll gestures. When you have both:
- A large navigation title that collapses on scroll
- `.tabBarMinimizeBehavior(.onScrollDown)` that collapses tab bar on scroll

These two scrolling behaviors conflict, with the navigation bar consuming the scroll gesture before the TabView can detect it for minimization.

**Solution implemented:**
Reverted HomeView navigation approach to match 815f1e2:
1. Changed `.navigationBarTitleDisplayMode(.large)` → `.navigationBarHidden(true)`
2. Removed `.navigationTitle("EchoCast")`
3. Removed entire `.toolbar { }` block
4. Removed `.toolbarBackground()` and `.preferredColorScheme()`
5. Added inline header HStack at top of ScrollView:
   - "EchoCast" title with `.largeTitleEcho()` font
   - Greeting text below
   - Magnifying glass and gear buttons inline on right
   - Proper padding: `.horizontal` + `.top`

**Code changes:**

Removed navigation bar:
```swift
// REMOVED:
.navigationTitle("EchoCast")
.navigationBarTitleDisplayMode(.large)
.toolbarBackground(Color.echoBackground, for: .navigationBar)
.preferredColorScheme(.dark)
.toolbar { ... }
```

Added inline header:
```swift
// ADDED inside ScrollView > VStack:
```
